# 機能的コア / 命令的シェル

このラボでは、Antmanの記事の用語を、F#とScott Wlaschinのワークフロー設計に接続します。

## 機能的コア

- Entity: レコード、DU、シングルケースユニオン
- 不変条件: 一つのビジネスルールを判定する純粋関数
- Deriver: 必要なデータをすべて受け取り、変更またはドメインの結果を返す純粋関数
- 同じ入力には常に同じ出力
- DB、クロック、ランダム、ネットワークを直接呼び出さない

## 命令的シェル

- コントローラー: データ取得、Deriver呼び出し、結果に応じた効果の実行
- リポジトリ: DB表現とドメイン型の変換境界
- アダプター: 支払い、メール、キュー、HTTPなどの具体的実装
- コンポジションルート: 依存関係を組み立てる唯一の場所

## 判断基準

「この条件で承認できるか」はコアの責任です。「承認結果をDBへ保存する」はシェルの責任です。「DB接続が失敗した」はドメインの結果ではなくシステム障害です。

## 出力

`UpgradeSubscriptionController.create`の依存関係を一つずつフェイクに差し替え、各結果で呼ばれる効果だけをテストします。Deriverのビジネスルールはコントローラーテストで重複してテストしません。
