# 型によるドメインモデリング

## 基本対応

| ドメイン概念 | F#表現 |
|---|---|
| 複数項目をすべて持つ | レコード（AND型） |
| 複数状態のどれか | 区別付き共用体（OR型） |
| 制約付きプリミティブ | プライベートシングルケース共用体 + スマートコンストラクタ |
| 状態遷移 | 入力状態と出力状態が異なる関数 |
| ビジネス操作 | 名前付き関数型 |
| 予期しない失敗 | `Result`またはドメイン固有の共用体 |

## 重要原則

`string`を受け取って毎回バリデーションするのではなく、バリデーション済みの型を作り、その型を受け取る関数では再検証を不要にします。

ただし、外部からのJSON、DBの行、CLI引数は信用できません。境界でパースし、ドメイン型へ変換します。これは「Parse, don't validate」と同じ方向性です。

## 出力

- `CustomerId`、`Money`、`CardToken`のスマートコンストラクタを書く
- `ParkingPublicationState`をDUにする
- `Closed`なのに`Published`でもある状態を表現不能にする
- 型だけを見てワークフローを説明する
